1 선생님 계정만 출석체크 가능(teacher/hakik)

1. 선생님은 1년 단위로 바뀐다.
2. 선생님 수 역시 변동된다.
3. 학생은 1년뒤 반이 바뀐다.
4. 반은 학년별일수도, 중등부/고등부일수도, 중고등부 연합일수도 있다.
5. 모든 데이터는 1년 단위로 갱신되어야 한다.
6. 출석체크를 완료하는 순간, Score table이 갱신된다.

default 설계.
    - 출석/성경/요절체크하기
    - 점수계산자동으로 해주기.
    - 반별 점수 그래프 그려주기.(출석/요절/성경읽기 선택하면 각각으로도 보여줄수있도록)
        - 그래프 보여주는 방식은 한달 단위 / 한주한주 누적되는 형태.

DB 설계.
Class
    - 반ID
    - 날짜(yyyy.MM.dd)
    - 담임ID
    - 반이름
    -

Teacher
    - 담임ID
    - 이름
    - 생년월일
    - 연락처
    - 반ID

Student
    - 학생ID(숫자)
    - 학생이름
    - 생년월일
    - 학년
    - 반ID
    - 연락처
    - 등록일
    - 등반일
    - 메모

    add - 세례(입교)일, 학습일


Rollbook
    복합키 날짜 + 학생
    - 날짜(yyyy.mm.dd)
    - 학생ID
    - 학생이름
    - 반ID
    - 출석(출석/지각/결석(default))
    - 성경읽기
    - 요절(true/false)
    - 새친구여부(true/false)

class_score_history
    - 날짜
    - 반
    - 출석인원
    - 새친구출석인원
    - 출석점수(출석:10 / 지각:5 -> 출석인원-새친구출석인원)     (attendance)
    - 성경읽기점수(총 성경 읽은 장수/출석인원-새친구출석인원)            (reading)
    - 요절점수(총 요절 외운 인원/출석인원-새친구출석인원)          (memorize)
    - 출석랭킹스코어       (attendance_rank_score)
    - 요절랭킹스코어       (memorize_rank_score)
    - 성경읽기랭킹스코어   (reading_rank_scroe)
    - 특별점수(전도/등반 등) (special_score)
